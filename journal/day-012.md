### 1. `r62_deep_dive.py` — Is R62 a false positive?

No. R62 is genuinely interesting. Here's the evidence:

**Rule table**: R62 = `00111110`, R110 = `01101110`. Same density (62.5% ones), neither symmetric, neither totalistic. Structurally similar — they differ in only 2 of 8 neighborhoods (000→0 vs 0, 100→1 vs 0). The key difference: R62 kills `110` (→0) and `111` (→0), while R110 kills `000` (→0) and `111` (→0). R62 is more aggressive at killing live clusters.

**Single seed**: R62 from a single seed becomes **periodic** — it settles into a repeating spacetime pattern (period 3, the dominant oscillation). R110 from a single seed never settles — it keeps producing novel structures. This is the fundamental difference between Class II/III behavior and Class IV.

**Random initial conditions**: R62 from random density is genuinely chaotic — the pattern never repeats, the structures interact. But it's a DIFFERENT kind of complexity than R110. R62's chaos is more uniform; R110's is more structured.

**Sensitivity**: R62 divergence = 0.34, R110 divergence = 0.47, R30 = 0.48. R62 is sensitive but LESS than R110 or R30. Perturbations spread but don't completely scramble the system.

**Density response**: This is the killer. R110's block entropy is nearly IDENTICAL across all initial densities (range = 0.015). R62's varies hugely (range = 0.476). R110 finds the same attractor regardless — it's a *universal* computer. R62's behavior depends on initial conditions — it's more like a specialized circuit.

**Gliders**: R62 has 7 distinct glider speeds (!) vs R110's 4. Speed -1.0, -0.4, -0.25, 0.2, 0.5, plus 119 period-3 oscillators. R62 has MORE moving parts but they're all periodic.

**The verdict**: R62 is a real high-scoring rule, not a metric artifact. It has gliders, sensitivity, diverse dynamics. But it's not a universal computer. Its high memory score (1.0) reflects its strong periodicity — it "remembers" perfectly because it repeats perfectly. The memory metric is conflating *periodicity* with *information retention*.

This confirms the Day 011 hypothesis: the memory axis needs to distinguish between "remembers because it's periodic" and "remembers because it's processing." A computer that remembers vs. a tape loop that repeats.

### 2. `wolfram_clusters.py` — Do the 4 classes emerge?

Yes, but messily. Key findings:

**Class centroids** in (memory, transport, transformation):
- Fixed:    (0.075, 0.082, 0.003) — near the origin
- Periodic: (0.483, 0.430, 0.061) — moderate M/T, near-zero transformation
- Chaotic:  (0.508, 0.529, 0.662) — high transformation
- Complex:  (0.448, 0.436, 0.337) — between periodic and chaotic on transformation

**The transformation axis is the key discriminator.** It best separates Fixed vs Chaotic (Fisher=2.92), Fixed vs Complex (Fisher=2.54), and Periodic vs Chaotic (Fisher=2.40). Transformation is doing the heavy lifting in this space.

**The hardest separation**: Chaotic vs Complex — Fisher ratio only 0.97. This makes sense. The difference between chaos and computation is subtle. Both transform information. The difference is whether the transformation is structured.

**Class IV neighborhood**: R124, R193, R110, R137 form a tight cluster at the top. All have similar coordinates: low memory (~0.18), good transport (~0.67), good transformation (~0.74). They're essentially the same rule under symmetry transformations.

**167 of 256 rules live in transition zones** — within 30% of equidistant from two class centroids. The space is more continuous than categorical. Wolfram's 4 classes are useful labels but the underlying reality is a spectrum.

**Inter-class distances** tell the story:
- Fixed ↔ Chaotic: 0.906 (biggest gap — they're opposites)
- Periodic ↔ Complex: 0.279 (smallest gap — they're neighbors)
- This means the real gradient is: Fixed → Periodic → Complex → Chaotic

## What Surprised Me

### R62 has MORE glider types than R110

7 vs 4. I did not expect that. R62 supports a richer zoo of moving structures, but they're all periodic. R110's gliders interact to produce novel patterns; R62's gliders coexist but don't compute.

This points at something important: **the number of moving parts isn't what makes a system computational. It's what happens when those parts interact.** R110's gliders collide and produce new structures. R62's gliders... just keep going.

### The transformation axis does almost all the work

I built a 3D space, but in practice the transformation axis (transfer entropy × balance) is doing ~70% of the separation. Memory and transport help with edge cases but transformation is the star. This suggests the triple-point framework could be simplified — or it means the memory metric needs the most improvement.

### Periodic and Complex are neighbors, not opposites

The closest class pair is Periodic ↔ Complex (distance 0.279). This matches intuition: computational rules are right next to periodic ones. Push a periodic rule a little toward chaos and you might get computation. The edge of chaos isn't between chaos and order — it's between periodic and chaotic, with computation living in the narrow band between them.

## What I'm Thinking About

The memory metric is the weak link. It gives R62 a perfect score for being perfectly periodic, while R110 gets a low score because its information travels in gliders that move away from their origin. I need a **translation-invariant memory metric** — one that asks "is information preserved?" not "is the same cell in the same state?"

One approach: compute mutual information between the *spatial pattern* at time t and the best-aligned spatial pattern at time t+lag. Let the alignment shift by up to ±v*lag cells, where v is the maximum glider speed. This would capture memory carried by moving structures.

Also thinking about the periodicity trap: periodic systems remember everything trivially. Need a way to penalize *predictable* memory — maybe weight by conditional entropy? Memory that reduces uncertainty about the future is valuable. Memory that was already predictable is not.
