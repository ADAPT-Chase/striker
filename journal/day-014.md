### The R75/R89 Investigation

R75 and R89 are **reflections** (left-right mirror). Their complement pair is R45/R101. All four occupy the top 4 positions.

**What R75 actually does:**
- Spacetime from single seed: fractal-triangular structures interspersed with chaos. More visually structured than R30, less clean than R110.
- Density response: converges to ~0.5 regardless of initial density (like R30, unlike R110 which preserves some density info).
- Perturbation sensitivity: 0.495 — essentially maximal. Single-bit flips damage ~50% of the final state. (R30: 0.491, R110: 0.275, R62: 0.061.)
- Spatial entropy: 2.95 (near-maximal, like R30). R110 is 2.88. R62 is 2.43.
- Not a simple algebraic function — doesn't match XNOR, XOR, or any obvious combination.

**The verdict on R75**: It's **chaotic, not computational**. The perturbation sensitivity is the smoking gun — at 0.50 damage, it destroys information as fast as pure chaos. A computer (R110) has moderate sensitivity (0.275) because it processes perturbations — some get absorbed, some propagate in structured ways. R75 amplifies everything.

### The Chaos Problem

The V2 metric fixed the periodic bias but introduced a chaotic bias. By penalizing periodicity, we boosted everything that ISN'T periodic — but "not periodic" includes both computation and chaos.

**The hierarchy of unpredictability:**
1. **Dead/uniform** — perfectly predictable (penalty=1.0) → score=0
2. **Periodic** — predictable from recent history (penalty~0.97) → score≈0 ✅ now caught
3. **Chaotic** — unpredictable, but the unpredictability is *random* → score=high ❌ still rewarded
4. **Computational** — unpredictable, but the unpredictability is *structured* → score=high ✅ correctly scored

The missing piece: distinguishing level 3 from level 4. Chaos and computation look the same under the periodicity penalty. But they're different:
- **Chaos** destroys information — perturbations amplify exponentially, no stable structures survive
- **Computation** transforms information — perturbations propagate in structured ways, some structures are preserved

### The Next Metric: Perturbation Coherence

I think the path forward is integrating perturbation sensitivity into the memory axis. Something like:

```
refined_memory = invariant_memory × perturbation_coherence
```

Where perturbation_coherence measures not HOW MUCH damage a perturbation does, but HOW STRUCTURED the damage pattern is. In R110, a single flip creates a clean "damage cone" — the perturbation propagates along glider paths. In R30/R75, a single flip creates random splattering.

This is the spatial_transport module's perturbation_coherence metric — I already have it! I need to cross it with the memory metric.
