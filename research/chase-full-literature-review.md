# Literature Review: Information Transformation & Causal Flow in Collective Computation

**Target Problem:** In collective computation simulations, the "transformation" axis (how information mutates during spatial propagation between Novas) bottlenecks the system, despite highly optimized memory and transport layers.

## 1. Key Papers & Findings

### A. A Quantifiable Information-Processing Hierarchy Provides a Necessary Condition for Detecting Agency
* **Authors/Date:** (arXiv: 2601.03498v1, Jan 2026)
* **Link:** https://arxiv.org/abs/2601.03498
* **Main Finding:** Proposes a bottom-up framework for information-processing "orders" in agents. Class I (reactive) and Class II (memory-bound) systems follow fixed transformation rules. Class III (adaptive) systems possess transformation rules that themselves change as a function of prior activity.
* **Relevance:** Fixed-rule transformations create bottlenecks. Adaptive transformation is required.

### B. Directional Information Transfer Between Interacting Brownian Particles
* **Authors/Date:** (arXiv: 2603.10475, Mar 2026)
* **Link:** https://arxiv.org/abs/2603.10475
* **Main Finding:** Transfer entropy reveals directional flow driven by asymmetry.
* **Relevance:** Introduce asymmetric weighting to reduce redundant bidirectional processing.

### C. Long-Term Prediction on Graph Data With Causal Network Construction
* **Authors/Date:** IEEE TAI (2024)
* **Link:** https://www.computer.org/csdl/journal/ai/2024/07/10382474/1TxRPcKNqTK
* **Main Finding:** Uses conditional entropy to identify true causal parents.
* **Relevance:** Prune non-causal transformations.

## 2. Actionable Techniques

### 1. Conditional Transfer Entropy Pruning
Drop payloads if they do not reduce uncertainty.

### 2. Asymmetric Payload Weighting
Assign "mass" to events → directional processing.

### 3. Operator-Based Transformation
Use compiled Wasm operators instead of raw parsing.

## 3. Assessment

- Prioritize selective processing (CTE)
- Enforce directional flow (mass)
- Move to adaptive transforms (Class III)
